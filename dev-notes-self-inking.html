What this document covers

Overview

This note covers the full implementation of the revamped stamps.net.au/self-inking-rubber-stamps pillar page — the site's #1 ranking page and primary revenue driver. The revamp transforms a single-brand DF category page into a multi-brand pillar hub serving DF, Trodat, Colop, and Trodat Multi-Colour.

Each section shows the Existing Code (extracted from the live page HTML) alongside the Proposed Code from the staging file. Every change includes its SEO purpose and a go-live checklist item. The staging file is self-inking-rubber-stamps.html.

Critical rule: All product HTML is pre-rendered server-side in the DOM. No JS-rendered products. No lazy-loaded tabs. CSS class toggle only for brand switching.

Critical Technical Decision

Product Grid — DOM Rendering Strategy

This is the most important technical decision on the page. How products are rendered determines whether Google can index them — and therefore whether they contribute to rankings at all.

📈 SEO Purpose — Three options, only one is correct
  • Option A — JS-rendered (DO NOT USE): Products injected by JavaScript after page load. Googlebot may not execute JS on first crawl pass. Products invisible to search engine = zero indexation value.
  • Option B — Lazy load per tab click (DO NOT USE): Products only fetch when a tab is clicked. Google never clicks. Structured data referencing these products would also be invalid since the content doesn't exist in the HTML.
  • Option C — Static DOM + CSS toggle (PROPOSED ✓): All 115 products are server-rendered in the HTML on page load. CSS display:none hides inactive brand panels visually. Googlebot reads all 115 product names, alt texts, and URLs on first crawl. switchBrand() only adds/removes a CSS class — no fetching, no innerHTML injection.

Accessibility + Image SEO

Product Image Alt & Title Attributes

The existing page has 33 product images with no alt text (verified by audit) and 0 title attributes. This is a WCAG 2.1 Level A accessibility failure and a significant Image Search ranking opportunity being missed.

📈 SEO Purpose
  • alt attribute: Primary signal for Google Image Search ranking. Also fallback text for screen readers and when image fails to load.
  • title attribute: Tooltip on hover; additional keyword signal; standard ecommerce practice on product listings.
  • "from $X" convention: Matches Google Shopping, Amazon, and eBay standard. "Starts at" implies a configurable floor price. "Starts from" is grammatically awkward. Use "from $X".
  • width + height: Adding explicit dimensions prevents CLS (Cumulative Layout Shift) — a Core Web Vital metric that directly affects ranking.

E-E-A-T + Rich Results

Reviews & FAQ Sections

Neither a reviews section nor a FAQ exists on the current live page. These two additions target different SEO goals: reviews build E-E-A-T and trust signals; FAQ targets rich results and People Also Ask capture.

📈 SEO Purpose
  • Reviews (E-E-A-T): Real named reviews with dates are stronger E-E-A-T signals than anonymous testimonials. Aggregate 5.0 score feeds the LocalBusiness JSON-LD aggregateRating field, supporting rich result eligibility. Connect to the live review database for production — do not hardcode.
  • FAQPage JSON-LD: Enables Google's FAQ rich result. Can appear as an expandable block below the main SERP listing, effectively doubling page real estate in search results. FAQ answers are also cited frequently in AI Overviews for branded/comparison queries.
  • FAQ accordion is CSS max-height: Answer content is in the DOM at all times (Google reads it). The accordion uses max-height:0 → 350px transition — not display:none on answers — ensuring full crawlability.

Mobile-First Indexing

Mobile Responsive — 5 Breakpoints

Google uses mobile-first indexing. The mobile version of this page is what Google primarily indexes and ranks. All 5 breakpoints ensure no content is hidden, collapsed, or overflow-clipped on any screen size.

📈 SEO Purpose
  • Mobile-first indexing means broken mobile layouts directly impact rankings. Content that overflows or is visually broken on mobile is still crawled, but user signals (bounce rate, dwell time) will suffer.
  • The brand strip uses overflow-x:auto; scrollbar-width:none on mobile to allow horizontal scrolling without the strip breaking layout.
  • CTA buttons stack full-width at 479px to maximise tap target size (minimum 44px per WCAG 2.1 guideline).
